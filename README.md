# S3 Клиент для 1С

Серверный общий модуль для работы с S3-совместимыми объектными хранилищами из 1С:Предприятия 8.3.

Проект содержит выгрузку конфигурации 1С в XML-формате. Основная логика находится в общем модуле `s3Клиент`.

## Возможности

- PUT Object: загрузка `ДвоичныеДанные` в bucket.
- GET Object: чтение объекта в `ДвоичныеДанные`.
- DELETE Object: удаление одного или нескольких объектов.
- HEAD Object: проверка существования и чтение технических заголовков.
- ListObjectsV2: получение списка объектов bucket.
- Presigned URL: временные ссылки на чтение и запись.
- AWS Signature Version 4 для S3-compatible API.
- Path-style и virtual-hosted-style адресация.

## Требования

- 1С:Предприятие 8.3.21 (Или переделайте ВызватьИсключение, для более старших версий платформы)
- Выполнение общего модуля на сервере.
- S3-совместимое хранилище: MinIO, Ceph Object Storage, AWS S3 или совместимый сервис.

```text

src/cf/CommonModules/s3Клиент/Ext/Module.bsl Основной код S3-клиента
```

Локальные служебные папки, результаты сборки и бинарные инструменты не являются частью библиотеки и исключены из публикации.

## Быстрый старт

1. Загрузите XML-выгрузку из `src/cf` в тестовую информационную базу или перенесите общий модуль `s3Клиент` в свою конфигурацию.
2. Подготовьте S3-compatible bucket. Для локальной разработки можно использовать MinIO: [docs/minio-s3client-dev.md](docs/minio-s3client-dev.md).
3. Создайте параметры подключения и вызовите нужную экспортную функцию.

```bsl
ПараметрыS3 = s3Клиент.ПараметрыПодключения(
	"http://localhost:9000",
	"us-east-1",
	"s3client-dev",
	"minioadmin",
	"minioadmin123",
	Истина,
	Ложь);

Данные = ПолучитьДвоичныеДанныеИзСтроки("hello from 1C", КодировкаТекста.UTF8, Ложь);
s3Клиент.ЗагрузитьДвоичныеДанные(ПараметрыS3, "examples/hello.txt", Данные, "text/plain");

ПрочитанныеДанные = s3Клиент.ПрочитатьДвоичныеДанные(ПараметрыS3, "examples/hello.txt");
Информация = s3Клиент.ПолучитьИнформациюОбОбъекте(ПараметрыS3, "examples/hello.txt");
Список = s3Клиент.ПолучитьСписокОбъектов(ПараметрыS3, "examples/");

s3Клиент.УдалитьОбъект(ПараметрыS3, "examples/hello.txt");
```

Учетные данные в примере подходят только для локального MinIO из документации. Не используйте их в production.

## Публичный API

| Функция | Назначение |
| --- | --- |
| `ПараметрыПодключения` | Создает структуру параметров подключения. |
| `ЗагрузитьДвоичныеДанные` | Загружает объект через PUT Object. |
| `ПрочитатьДвоичныеДанные` | Читает объект через GET Object. |
| `УдалитьОбъект` | Удаляет один объект. |
| `УдалитьСписокОбъектов` | Последовательно удаляет список объектов. |
| `ОбъектСуществует` | Проверяет существование объекта через HEAD Object. |
| `ПолучитьИнформациюОбОбъекте` | Возвращает заголовки и техническую информацию объекта. |
| `ПолучитьСписокОбъектов` | Возвращает результат ListObjectsV2. |
| `ПолучитьВременнуюСсылкуНаЧтение` | Создает presigned GET URL. |
| `ПолучитьВременнуюСсылкуНаЗапись` | Создает presigned PUT URL. |

### Параметры подключения

```bsl
ПараметрыS3 = s3Клиент.ПараметрыПодключения(
	Endpoint,
	Region,
	Bucket,
	AccessKeyId,
	SecretAccessKey,
	PathStyle,
	UseSSL,
	Таймаут);
```

Параметры:

- `Endpoint` - адрес S3 API без имени bucket, например `https://s3.example.com` или `http://localhost:9000`.
- `Region` - регион для AWS Signature V4. Для MinIO обычно используется `us-east-1`.
- `Bucket` - имя bucket.
- `AccessKeyId` - идентификатор ключа доступа.
- `SecretAccessKey` - секретный ключ доступа.
- `PathStyle` - `Истина` для адресации `endpoint/bucket/key`, `Ложь` для `bucket.endpoint/key`.
- `UseSSL` - `Истина` для HTTPS, `Ложь` для HTTP. Если схема указана в `Endpoint`, она имеет приоритет.
- `Таймаут` - таймаут HTTP-соединения в секундах, по умолчанию `30`.

### Загрузка объекта

```bsl
Результат = s3Клиент.ЗагрузитьДвоичныеДанные(
	ПараметрыS3,
	"documents/report.pdf",
	ДвоичныеДанные,
	"application/pdf");
```

Возвращает структуру:

- `КодСостояния` - HTTP-код ответа. Для успешной загрузки ожидается `200`.
- `Заголовки` - заголовки HTTP-ответа.

Если `ТипСодержимого` не передан, модуль пытается подобрать `Content-Type` по расширению ключа.

### Чтение и проверка объекта

```bsl
Если s3Клиент.ОбъектСуществует(ПараметрыS3, "documents/report.pdf") Тогда
	Информация = s3Клиент.ПолучитьИнформациюОбОбъекте(ПараметрыS3, "documents/report.pdf");
	Данные = s3Клиент.ПрочитатьДвоичныеДанные(ПараметрыS3, "documents/report.pdf");
КонецЕсли;
```

`ПрочитатьДвоичныеДанные` возвращает `ДвоичныеДанные`.

`ПолучитьИнформациюОбОбъекте` возвращает структуру с техническими данными объекта:

- `Bucket` - имя bucket.
- `Ключ` - ключ объекта.
- `Размер` - размер объекта в байтах.
- `ETag` - значение ETag, если провайдер его вернул.
- `LastModified` - строковое значение заголовка `Last-Modified`.
- `ContentType` - тип содержимого.
- `Заголовки` - все заголовки HTTP-ответа.

Если объект не найден, `ПолучитьИнформациюОбОбъекте` возвращает `Неопределено`, а `ОбъектСуществует` возвращает `Ложь`.

### Список объектов

```bsl
Список = s3Клиент.ПолучитьСписокОбъектов(
	ПараметрыS3,
	"documents/",
	100,
	ТокенПродолжения);
```

Возвращает структуру:

- `Объекты` - массив структур с полями `Ключ`, `Размер`, `ETag`, `LastModified`.
- `Префиксы` - массив общих префиксов, если они есть в ответе S3.
- `СледующийТокенПродолжения` - токен следующей страницы.
- `ЕстьСледующаяСтраница` - `Истина`, если список можно продолжить.

Для первой страницы передайте пустой `ТокенПродолжения` или не передавайте его. Для следующей страницы используйте `СледующийТокенПродолжения` из предыдущего ответа.

### Удаление объектов

```bsl
Результат = s3Клиент.УдалитьОбъект(ПараметрыS3, "documents/report.pdf");
```

Успешное удаление возвращает структуру с `КодСостояния = 204`.

Для нескольких объектов:

```bsl
Ключи = Новый Массив;
Ключи.Добавить("documents/a.txt");
Ключи.Добавить("documents/b.txt");

Результаты = s3Клиент.УдалитьСписокОбъектов(ПараметрыS3, Ключи);
```

`УдалитьСписокОбъектов` выполняет последовательные DELETE-запросы и возвращает массив результатов по ключам.

### Временные ссылки

```bsl
СсылкаНаЧтение = s3Клиент.ПолучитьВременнуюСсылкуНаЧтение(
	ПараметрыS3,
	"documents/report.pdf",
	3600);

СсылкаНаЗапись = s3Клиент.ПолучитьВременнуюСсылкуНаЗапись(
	ПараметрыS3,
	"uploads/new-file.txt",
	900);
```

Срок действия задается в секундах. Допустимый диапазон: от `1` до `604800` секунд.

### Ошибки

При ошибках HTTP или S3 модуль вызывает исключение 1С. Текст исключения начинается с `S3:` и, если доступно, содержит HTTP-код, код ошибки S3 и тело ответа. Не записывайте в журналы реальные `SecretAccessKey`, production bucket и внутренние endpoint.

## Ограничения

Пока не реализованы multipart upload, session token, retries/backoff, server-side encryption, Object Lock, ACL, tags, metadata headers и CopyObject. Ошибки S3 возвращаются как исключения 1С с HTTP-кодом, кодом S3 и телом ответа, если оно доступно.

Разбор XML-ответов S3 выполнен легковесными строковыми функциями. Для базовых ответов S3/MinIO этого достаточно, но при расширении API стоит перейти на штатный XML-парсер 1С.

## Проверка

Локальная проверка XML-выгрузки без запуска 1С:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts/validate-cf.ps1
```

Интеграционный smoke-test описан в [tests/minio-smoke.bsl](tests/minio-smoke.bsl). Он рассчитан на локальный MinIO из [docs/minio-s3client-dev.md](docs/minio-s3client-dev.md) и проверяет загрузку, чтение, HEAD, список объектов, presigned URL и удаление объекта.

Сценарий можно выполнить из временной обработки, консоли кода или собственной тестовой обработки в базе, где загружен общий модуль `s3Клиент`.

## Релизы

Основной способ распространения - исходники в `src/cf`. Пользователь может загрузить XML-выгрузку в тестовую базу или перенести общий модуль `s3Клиент` в свою конфигурацию.

Собранные файлы `.cf`, `.epf` или `.erf` не хранятся в основной ветке. Если проекту понадобится готовый артефакт для конкретной версии, его можно прикладывать к GitHub Releases отдельно от исходников.
